# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 2.x | Yes |
| < 2.0 | No |

## Reporting a Vulnerability

Report vulnerabilities through GitHub Security Advisories:

https://github.com/gongahkia/tanabata/security/advisories/new

Do not open a public issue for suspected vulnerabilities.

## Disclosure Process

Tanabata follows a 90-day coordinated disclosure window from the date a report is accepted as in scope. Fixes may be released sooner when a validated issue has a clear remediation.

## Scope

In scope:

- Production builds of Tanabata.
- Vulnerabilities in the API, catalog storage, ingestion flow, container image, and release artifacts.

Out of scope:

- Local development bypasses.
- Vulnerabilities that require direct write access to a developer machine.
- Third-party service outages or rate limits without a Tanabata security impact.

PGP-encrypted email is not offered unless a maintained key is published here later.
